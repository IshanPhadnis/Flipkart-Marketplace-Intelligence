# Q1. Monthly Revenue Trend

## 🎯 Objective
Analyze the total revenue generated each month from delivered orders to identify sales trends over time.

## 💼 Business Value
Monthly revenue trends help businesses:
- Track overall sales performance.
- Identify seasonality.
- Evaluate marketing campaigns.
- Forecast future revenue.

## 🧠 Approach
- Filter only delivered orders.
- Group orders by month.
- Calculate total revenue.
- Sort chronologically.

## 💻 SQL

```sql
SELECT
    FORMAT_DATE('%Y-%m', order_date) AS month,
    SUM(amount) AS total_revenue
FROM flipkart.orders
WHERE LOWER(status) = 'delivered'
GROUP BY month
ORDER BY month;
```

## 📝 Explanation

- `FORMAT_DATE('%Y-%m')` extracts the year and month.
- `SUM(amount)` calculates total revenue.
- `LOWER(status)` ensures inconsistent capitalization doesn't affect the analysis.

## 💡 Business Recommendation

Use monthly revenue trends to identify peak sales periods and optimize marketing campaigns, inventory planning, and staffing.

# Q2. Orders by Weekday

## 🎯 Objective

Determine which day of the week receives the highest number of customer orders.

## 💼 Business Value

Knowing which weekdays generate the highest order volume helps businesses:

- Plan staffing.
- Schedule promotions.
- Improve warehouse operations.
- Optimize delivery logistics.

## 🧠 Approach

- Extract weekday from order date.
- Count orders for each weekday.
- Sort from highest to lowest.

## 💻 SQL

```sql
SELECT
    FORMAT_DATE('%A', order_date) AS weekday,
    COUNT(order_id) AS total_orders
FROM flipkart.orders
GROUP BY weekday
ORDER BY total_orders DESC;
```

## 📝 Explanation

- `FORMAT_DATE('%A')` extracts the weekday.
- `COUNT(order_id)` counts the number of orders placed on each day.

## 💡 Business Recommendation

Run promotional campaigns on low-performing weekdays and ensure operational capacity on peak shopping days.

# Q3. Monthly Average Order Value (AOV)

## 🎯 Objective

Calculate the Average Order Value (AOV) for each month.

## 💼 Business Value

Average Order Value helps businesses understand customer spending behavior and evaluate the effectiveness of upselling and cross-selling strategies.

## 🧠 Approach

- Filter delivered orders.
- Group data by month.
- Calculate the average order amount.

## 💻 SQL

```sql
SELECT
    FORMAT_DATE('%Y-%m', order_date) AS month,
    ROUND(AVG(amount),2) AS average_order_value
FROM flipkart.orders
WHERE LOWER(status) = 'delivered'
GROUP BY month
ORDER BY month;
```

## 📝 Explanation

- `AVG(amount)` calculates the average spending per order.
- `ROUND()` improves readability.
- Grouping by month allows trend analysis.

## 💡 Business Recommendation

Monitor AOV over time to measure customer purchasing behavior. Increase AOV through bundles, discounts above a spending threshold, and personalized product recommendations.

# Q6. Revenue by City

## 🎯 Objective

Identify which cities generate the highest revenue.

## 💼 Business Value

City-level revenue analysis helps businesses:

- Identify high-performing markets.
- Allocate marketing budgets effectively.
- Plan warehouse expansion.
- Improve regional inventory management.

## 🧠 Approach

- Join the Customers and Orders tables.
- Filter delivered orders.
- Group by city.
- Calculate total revenue.
- Sort by revenue.

## 💻 SQL

```sql
SELECT
    c.city,
    SUM(o.amount) AS total_revenue
FROM flipkart.orders o
JOIN flipkart.customers c
    ON o.customer_id = c.customer_id
WHERE LOWER(o.status) = 'delivered'
GROUP BY c.city
ORDER BY total_revenue DESC;
```

## 📝 Explanation

- The `JOIN` connects customer information with order details.
- `SUM(amount)` calculates revenue generated from each city.
- Only delivered orders are considered.

## 💡 Business Recommendation

Invest more in high-revenue cities while analyzing low-performing regions for growth opportunities through localized marketing campaigns.

# Q7. Repeat Customers

## 🎯 Objective

Identify customers who have placed more than one delivered order.

## 💼 Business Value

Repeat customers are one of the strongest indicators of customer loyalty and business health.

This analysis helps businesses:

- Measure customer retention.
- Identify loyal customers.
- Design loyalty and rewards programs.
- Increase Customer Lifetime Value (CLV).

## 🧠 Approach

- Filter delivered orders.
- Group orders by customer.
- Count the number of delivered orders.
- Keep only customers with more than one order.

## 💻 SQL

```sql
SELECT
    customer_id,
    COUNT(order_id) AS total_orders
FROM flipkart.orders
WHERE LOWER(status) = 'delivered'
GROUP BY customer_id
HAVING COUNT(order_id) > 1
ORDER BY total_orders DESC;
```

## 📝 Explanation

- `COUNT(order_id)` counts the number of successful purchases.
- `HAVING` filters customers after aggregation.
- Customers with more than one delivered order are classified as repeat customers.

## 💡 Business Recommendation

Reward repeat customers with loyalty programs, exclusive offers, and personalized recommendations to improve retention and maximize long-term revenue.

# Q9. Month-over-Month (MoM) Revenue Growth

## 🎯 Objective

Calculate the month-over-month percentage change in revenue to measure business growth over time.

## 💼 Business Value

Month-over-Month Growth helps businesses:

- Monitor sales performance.
- Detect growth or decline trends.
- Evaluate the success of promotions.
- Forecast future revenue.

## 🧠 Approach

- Calculate monthly revenue.
- Use the `LAG()` window function to retrieve the previous month's revenue.
- Compute the percentage growth between consecutive months.

## 💻 SQL

```sql
WITH monthly_revenue AS (
    SELECT
        FORMAT_DATE('%Y-%m', order_date) AS month,
        SUM(amount) AS total_revenue
    FROM flipkart.orders
    WHERE LOWER(status) = 'delivered'
    GROUP BY month
)

SELECT
    month,
    total_revenue,
    LAG(total_revenue) OVER (ORDER BY month) AS previous_month_revenue,
    ROUND(
        ((total_revenue - LAG(total_revenue) OVER (ORDER BY month))
        / LAG(total_revenue) OVER (ORDER BY month)) * 100,
        2
    ) AS mom_growth_percentage
FROM monthly_revenue
ORDER BY month;
```

## 📝 Explanation

- `LAG()` retrieves the previous month's revenue.
- The growth percentage is calculated using the standard percentage growth formula.
- The first month returns `NULL` because there is no previous month for comparison.

## 💡 Business Recommendation

Investigate months with declining revenue to identify potential operational issues, seasonal trends, or ineffective marketing campaigns. Replicate strategies from months showing strong growth.

# Q10. Rolling 3-Month Revenue Average

## 🎯 Objective

Calculate the rolling 3-month average revenue to smooth short-term fluctuations and identify long-term sales trends.

## 💼 Business Value

Monthly revenue can fluctuate due to promotions, holidays, or seasonal demand. A rolling average helps businesses:

- Identify long-term sales trends.
- Reduce the impact of short-term spikes.
- Improve forecasting accuracy.
- Support strategic business planning.

## 🧠 Approach

- Calculate total revenue for each month.
- Use a window function to average the current month and the previous two months.
- Display the rolling average alongside monthly revenue.

## 💻 SQL

```sql
WITH monthly_revenue AS (
    SELECT
        FORMAT_DATE('%Y-%m', order_date) AS month,
        SUM(amount) AS total_revenue
    FROM flipkart.orders
    WHERE LOWER(status) = 'delivered'
    GROUP BY month
)

SELECT
    month,
    total_revenue,
    ROUND(
        AVG(total_revenue) OVER (
            ORDER BY month
            ROWS BETWEEN 2 PRECEDING AND CURRENT ROW
        ),
        2
    ) AS rolling_3_month_average
FROM monthly_revenue
ORDER BY month;
```

## 📝 Explanation

- The CTE calculates total revenue for each month.
- The window function considers the current month and the previous two months.
- `ROWS BETWEEN 2 PRECEDING AND CURRENT ROW` creates a moving window of three months.
- `AVG()` computes the rolling average, while `ROUND()` improves readability.

## 💡 Business Recommendation

Use the rolling average instead of monthly revenue alone when forecasting future sales, evaluating long-term growth, and making inventory or staffing decisions.

# Q11. Customer Segmentation based on Customer Lifetime Value (CLV)

## 🎯 Objective

Segment customers into High, Medium, and Low Value categories based on their Customer Lifetime Value (CLV).

## 💼 Business Value

Not all customers contribute equally to a business. Customer segmentation helps companies:

- Identify VIP customers.
- Personalize marketing campaigns.
- Improve customer retention.
- Allocate resources more effectively.

## 🧠 Approach

- Calculate each customer's total spending (CLV).
- Use a `CASE` statement to assign customers to different value segments.
- Sort customers by CLV.

## 💻 SQL

```sql
WITH customer_clv AS (
    SELECT
        c.customer_id,
        c.customer_name,
        COUNT(o.order_id) AS total_orders,
        SUM(o.amount) AS customer_lifetime_value
    FROM flipkart.customers c
    JOIN flipkart.orders o
        ON c.customer_id = o.customer_id
    WHERE LOWER(o.status) = 'delivered'
    GROUP BY c.customer_id, c.customer_name
)

SELECT
    customer_id,
    customer_name,
    total_orders,
    customer_lifetime_value,
    CASE
        WHEN customer_lifetime_value > 60000 THEN 'High-Value'
        WHEN customer_lifetime_value BETWEEN 40000 AND 60000 THEN 'Medium-Value'
        ELSE 'Low-Value'
    END AS customer_segment
FROM customer_clv
ORDER BY customer_lifetime_value DESC;
```

## 📝 Explanation

- The CTE calculates each customer's total orders and lifetime spending.
- The `CASE` statement classifies customers into predefined value segments.
- Results are ordered by CLV so the highest-value customers appear first.

## 💡 Business Recommendation

High-value customers should receive VIP benefits, personalized recommendations, and exclusive offers. Medium-value customers can be nurtured through targeted promotions, while low-value customers can be encouraged to increase spending through discounts and loyalty programs.

# Q12. Rank Customers Based on Customer Lifetime Value (CLV)

## 🎯 Objective

Rank customers based on their total spending (Customer Lifetime Value) to identify the highest-value customers.

## 💼 Business Value

Ranking customers by their lifetime spending helps businesses:

- Identify VIP customers.
- Reward loyal customers with exclusive benefits.
- Prioritize customer retention efforts.
- Allocate marketing budgets more effectively.
- Improve personalized recommendation strategies.

## 🧠 Approach

- Join the Customers, Orders, and Order Items tables.
- Filter only delivered orders.
- Calculate the Customer Lifetime Value (CLV) by summing the value of all purchased products.
- Count the total number of orders placed by each customer.
- Use the `DENSE_RANK()` window function to rank customers based on their CLV.

## 💻 SQL

```sql
WITH customer_clv AS (
    SELECT
        o.customer_id,
        CONCAT(c.first_name, ' ', c.last_name) AS full_name,
        COUNT(DISTINCT o.order_id) AS total_orders,
        SUM(oi.price * oi.quantity) AS customer_lifetime_value
    FROM flipkart.customers c
    JOIN flipkart.orders o
        ON c.customer_id = o.customer_id
    JOIN flipkart.order_items oi
        ON o.order_id = oi.order_id
    WHERE LOWER(o.status) = 'delivered'
    GROUP BY
        o.customer_id,
        CONCAT(c.first_name, ' ', c.last_name)
)

SELECT
    customer_id,
    full_name,
    total_orders,
    customer_lifetime_value,
    DENSE_RANK() OVER (ORDER BY customer_lifetime_value DESC) AS customer_rank
FROM customer_clv
ORDER BY customer_rank;
```

## 📝 Explanation

- The CTE calculates each customer's total lifetime spending.
- `SUM(oi.price * oi.quantity)` computes the total value of all products purchased by the customer.
- `COUNT(DISTINCT o.order_id)` counts the number of unique delivered orders, preventing duplicate counts caused by joining with the `order_items` table.
- `DENSE_RANK()` assigns the same rank to customers with equal CLV while ensuring there are no gaps in the ranking sequence.
- Results are ordered from the highest-spending customer to the lowest.

## 💡 Business Recommendation

Businesses should focus on retaining top-ranked customers through VIP memberships, exclusive discounts, early product launches, and personalized marketing campaigns. Since acquiring new customers is typically more expensive than retaining existing ones, identifying high-value customers is essential for maximizing long-term profitability.

# Q13. Top 3 Customers in Each City

## 🎯 Objective

Identify the top 3 highest-value customers in each city based on their Customer Lifetime Value (CLV).

## 💼 Business Value

Knowing the highest-spending customers within each city helps businesses:

- Identify regional VIP customers.
- Launch city-specific loyalty programs.
- Personalize local marketing campaigns.
- Allocate sales and customer success resources efficiently.
- Understand regional purchasing behavior.

## 🧠 Approach

- Join the Customers, Orders, and Order Items tables.
- Filter only delivered orders.
- Calculate each customer's Customer Lifetime Value (CLV).
- Use the `DENSE_RANK()` window function with `PARTITION BY city` to rank customers **within each city**.
- Return only the top three customers from every city.

## 💻 SQL

```sql
WITH customer_clv AS (
    SELECT
        o.customer_id,
        c.city,
        CONCAT(c.first_name, ' ', c.last_name) AS full_name,
        SUM(oi.price * oi.quantity) AS customer_lifetime_value
    FROM flipkart.customers c
    JOIN flipkart.orders o
        ON c.customer_id = o.customer_id
    JOIN flipkart.order_items oi
        ON o.order_id = oi.order_id
    WHERE LOWER(o.status) = 'delivered'
    GROUP BY
        o.customer_id,
        c.city,
        CONCAT(c.first_name, ' ', c.last_name)
),

customer_rank AS (
    SELECT *,
           DENSE_RANK() OVER (
               PARTITION BY city
               ORDER BY customer_lifetime_value DESC
           ) AS customer_rank
    FROM customer_clv
)

SELECT *
FROM customer_rank
WHERE customer_rank <= 3
ORDER BY city, customer_rank;
```

## 📝 Explanation

- The first CTE calculates the Customer Lifetime Value (CLV) for every customer.
- `SUM(oi.price * oi.quantity)` calculates the total value of all products purchased by each customer.
- `DENSE_RANK()` ranks customers based on their CLV.
- `PARTITION BY city` restarts the ranking for every city, allowing each city to have its own top customers.
- Filtering with `customer_rank <= 3` returns only the top three customers from each city.
- Results are ordered by city and customer rank for easy interpretation.

## 💡 Business Recommendation

Instead of rewarding only the company's overall highest-spending customers, businesses should identify their top customers within each city. This enables localized marketing campaigns, city-wise loyalty programs, personalized customer engagement, and better regional sales strategies.

## 📚 Key SQL Concepts Learned

- Common Table Expressions (CTEs)
- Multi-table Joins
- Aggregate Functions (`SUM()`)

- `PARTITION BY`

# Q14. Percentage Contribution of Each Customer to Total Revenue

## 🎯 Objective

Calculate the percentage contribution of each customer to the company's total revenue based on their Customer Lifetime Value (CLV).

## 💼 Business Value

Understanding each customer's contribution to total revenue helps businesses:

- Identify high-value customers driving the majority of sales.
- Prioritize customer retention efforts.
- Design tier-based loyalty programs.
- Allocate marketing budgets more effectively.
- Perform revenue concentration and Pareto (80/20) analysis.

## 🧠 Approach

- Calculate the Customer Lifetime Value (CLV) for each customer.
- Use a window function to calculate the company's total revenue without collapsing the rows.
- Compute each customer's percentage contribution using the formula:

> **Revenue Contribution (%) = (Customer CLV / Total Company Revenue) × 100**

- Round the result to two decimal places for better readability.

## 💻 SQL

```sql
WITH total_spending AS (
    SELECT
        c.customer_id,
        CONCAT(c.first_name, ' ', c.last_name) AS full_name,
        SUM(o.amount) AS CLV
    FROM flipkart.orders o
    JOIN flipkart.customers c
        ON o.customer_id = c.customer_id
    GROUP BY
        c.customer_id,
        CONCAT(c.first_name, ' ', c.last_name)
),

revenue_summary AS (
    SELECT *,
           SUM(CLV) OVER () AS company_revenue
    FROM total_spending
)

SELECT *,
       ROUND((CLV / company_revenue) * 100, 2) AS revenue_percentage
FROM revenue_summary
ORDER BY revenue_percentage DESC;
```

## 📝 Explanation

- The first CTE calculates each customer's Customer Lifetime Value (CLV).
- `SUM(CLV) OVER()` computes the total revenue generated by all customers while keeping every customer row intact.
- The revenue contribution percentage is calculated by dividing each customer's CLV by the company's total revenue and multiplying by 100.
- `ROUND(..., 2)` formats the percentage to two decimal places.
- Results are sorted from the highest revenue contributor to the lowest.

## 💡 Business Recommendation

Businesses should closely monitor customers contributing the largest share of revenue, as losing even a few of these customers can significantly impact overall sales. These customers should be targeted with personalized experiences, exclusive offers, loyalty rewards, and proactive customer support to maximize retention and long-term profitability.

- Common Table Expressions (CTEs)
- Window Functions
- `SUM() OVER()`
- Aggregate Functions
- Business KPI Calculations
- Percentage Contribution Analysis
